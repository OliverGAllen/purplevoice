INPUT contract (how Hammerspoon will invoke this script in Plan 03):
  - Spawned via `hs.task.new("/Users/<user>/.local/bin/voice-cc-record", onExitCallback)` then `:start()`.
  - Hammerspoon sends SIGTERM via `task:terminate()` when user releases hotkey.
  - hs.task does NOT inherit a login shell PATH — `/opt/homebrew/bin` is NOT available. Therefore: this script MUST use absolute paths for every external binary (Pitfall 2, ROB-03).
  - Script may also be invoked manually from a Terminal: `~/.local/bin/voice-cc-record` then Ctrl-C to stop. Both invocation paths produce the same behaviour. (ROADMAP success criterion #2 — "the bash glue script can be invoked manually outside Hammerspoon and produces the same transcript on stdout".)

OUTPUT contract (what Hammerspoon will receive):
  - On success: transcript text on stdout (no trailing newline; use `printf "%s"`), exit code 0.
  - On any failure in Phase 1: non-zero exit. Phase 2 will introduce the semantic exit code registry (10 = TCC denial, 11 = binary missing, 12 = whisper failed). For Phase 1, just exit non-zero on error — Hammerspoon's onExit callback in Plan 03 will treat any non-zero as "no paste".

WAV file contract (CAP-04):
  - sox writes to /tmp/voice-cc/recording.wav (single working file, overwritten per invocation per CONTEXT.md D-03).
  - When SIGTERM arrives, sox MUST finalise the WAV header before exiting — otherwise whisper-cli will fail to parse it.
  - sox's default SIGTERM handler does this correctly; do NOT override the trap on SOX's PID with kill -KILL. Use kill -TERM (or just `kill $SOX_PID`).

Vocab file contract (TRA-03, D-08):
  - Read from `~/.config/voice-cc/vocab.txt` (created by Plan 01 setup.sh).
  - Pass via `--prompt "$(cat <path>)"`. If the file is absent or unreadable, pass `--prompt ""` (empty) and continue — vocab is enhancement, not critical-path.

Whisper invocation flags (ARCHITECTURE.md Pattern 2 + Pitfall 4 + CONTEXT.md "Claude's Discretion"):
  - `-m ~/.local/share/voice-cc/models/ggml-small.en.bin`
  - `--language en`                          (Pitfall 4 belt-and-braces; CONTEXT.md says always pass for .en models)
  - `--no-timestamps`                        (we want plain text, not [00:00.000 --> 00:01.500] markers)
  - `--prompt "$(cat ~/.config/voice-cc/vocab.txt)"`  (TRA-03)
  - `-otxt -of "${WAV%.wav}"`                (write to /tmp/voice-cc/recording.txt for read-back)
  - `-f "$WAV"`
  - **DO NOT pass `--vad`** — that is Phase 2 (TRA-04). Phase 1 deliberately omits hallucination guards.
  - **DO NOT pass `--translate`** — Pitfall 4.

Symlink contract:
  - `~/.local/bin/voice-cc-record` is a symlink to the repo file (D-03 + STACK.md install.sh pattern: `ln -sf "$(pwd)/voice-cc-record" "$HOME/.local/bin/voice-cc-record"`).
  - This way, edits to the repo file are immediately effective without re-running setup.

  - No EXIT trap WAV cleanup (ROB-04 → Phase 2)
  - No re-entrancy guard (ROB-01 → Phase 2 in Lua, not bash)
  - No semantic exit code registry (Phase 2)

    ===== voice-cc-record =====

    ```bash
    #!/usr/bin/env bash
    # voice-cc-record — Phase 1 spike bash glue
    # Lifecycle: spawned by Hammerspoon hs.task (or invoked manually);
    # records mic audio via sox until SIGTERM (or Ctrl-C); transcribes via
    # whisper-cli; prints transcript to stdout for caller to paste.
    #
    # Phase 1 deliberately OMITS: VAD, denylist, duration gate, TCC detection,
    # WAV cleanup trap, semantic exit codes. All deferred to Phase 2.

    set -euo pipefail

    # --- Absolute binary paths (Pitfall 2, ROB-03) -------------------------
    # hs.task does NOT include /opt/homebrew/bin in PATH on Apple Silicon.
    # Override via env vars if you're on Intel (/usr/local/bin) or custom install.
    SOX_BIN="${SOX_BIN:-/opt/homebrew/bin/sox}"
    WHISPER_BIN="${WHISPER_BIN:-/opt/homebrew/bin/whisper-cli}"

    # --- Paths (D-03 XDG layout) -------------------------------------------
    MODEL="${VOICE_CC_MODEL:-$HOME/.local/share/voice-cc/models/ggml-small.en.bin}"
    VOCAB_FILE="$HOME/.config/voice-cc/vocab.txt"
    WAV_DIR="/tmp/voice-cc"
    WAV="$WAV_DIR/recording.wav"

    mkdir -p "$WAV_DIR"

    # --- Read vocab if available (TRA-03) ----------------------------------
    # Tolerate missing/unreadable vocab.txt — vocab is enhancement, not critical.
    if [ -r "$VOCAB_FILE" ]; then
      VOCAB="$(cat "$VOCAB_FILE")"
    else
      VOCAB=""
    fi

    # --- transcribe() — the SOLE STT abstraction boundary (ARCHITECTURE.md Pattern 2) ---
    # v1: one-shot whisper-cli per utterance.
    # v1.1 (deferred): swap body to `curl 127.0.0.1:8080/inference -F file=@"$1"`.
    # NOTHING else in this script may invoke whisper-cli directly.
    transcribe() {
      local wav_path="$1"
      local out_base="${wav_path%.wav}"
      "$WHISPER_BIN" \
        -m "$MODEL" \
        --language en \
        --no-timestamps \
        --prompt "$VOCAB" \
        -otxt -of "$out_base" \
        -f "$wav_path" >/dev/null 2>&1
      cat "${out_base}.txt"
    }

    # --- Capture: sox in foreground; SIGTERM (from Hammerspoon task:terminate
    # or manual Ctrl-C → SIGINT, which sox also handles cleanly) finalises the
    # WAV header before sox exits. CAP-04. ----------------------------------
    # We launch sox in the background and wait for it so this script's process
    # group stays clean and a SIGTERM to this script propagates to sox via the
    # trap below.
    "$SOX_BIN" -d -r 16000 -c 1 -b 16 "$WAV" >/dev/null 2>&1 &
    SOX_PID=$!

    # When this script receives SIGTERM (Hammerspoon) or SIGINT (Ctrl-C),
    # forward to sox so it finalises the WAV cleanly before we proceed.
    trap 'kill -TERM "$SOX_PID" 2>/dev/null || true' TERM INT

    # Block until sox exits (after SIGTERM forward, or natural exit).
    # `wait` returns sox's exit code; we use `|| true` so set -e doesn't kill
    # us on the expected non-zero from being signalled.
    wait "$SOX_PID" || true

    # --- Reset trap so we don't re-signal during transcription -------------
    trap - TERM INT

    # --- Transcribe ---------------------------------------------------------
    TRANSCRIPT="$(transcribe "$WAV")"

    # --- Emit on stdout (no trailing newline — caller pastes verbatim) -----
    # Trim leading/trailing whitespace that whisper-cli often emits.
    # Phase 2 will add denylist + empty-drop; Phase 1 just trims and emits.
    TRANSCRIPT="$(printf "%s" "$TRANSCRIPT" | sed -e 's/^[[:space:]]*//' -e 's/[[:space:]]*$//')"
    printf "%s" "$TRANSCRIPT"
    ```
