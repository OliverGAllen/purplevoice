  1. **`SOX_SIGNALED` flag** — Plan 02-01's tightened `wait` pattern was treating every normal hotkey release (SIGTERM-induced sox exit) as failure → exit 12 on every press. Fixed by flagging the trap path so signal-induced exits aren't misread as real failures.
  2. **Amplitude-based TCC detection** — Plan 02-01 RESEARCH §4 assumed sox emits a `Permission denied` stderr on TCC mic deny. Sequoia 15.7.5 actually feeds sox a *silent zero-stream* — sox runs to completion, no error written, but the WAV is at the 16-bit dither floor (max amplitude 0.000031 vs ~0.008 for real mic). Detection switched to `sox -n stat` amplitude check; threshold 0.0001 cleanly separates the regimes.

<blockers>

- **STRATEGIC**: Koe (https://github.com/missuo/koe, https://koe.fm) is the same product as voice-cc but more polished. Comparison from this session:

| Dimension | voice-cc (current) | Koe |
|---|---|---|
| Architecture | bash + Hammerspoon Lua + sox + whisper.cpp | Native Objective-C + Rust core, MIT |
| Hotkey default | cmd+shift+e (user wanted to try Fn) | **Fn** (configurable), hold mode AND toggle mode |
| STT backends | whisper.cpp small.en only | Apple Speech (macOS 26+), MLX Qwen3-ASR, sherpa-onnx, Doubao/Qwen cloud |
| Streaming | one-shot record→transcribe→paste | **real-time interim text** in floating pill |
| Text correction | Whisper --prompt vocab biasing | full LLM post-processing (any OpenAI-compatible API or local MLX) |
| Privacy default | local-only always | defaults to Doubao cloud (must opt into local) |
| Footprint | ~190 MB models + 200 MB Hammerspoon dep | 15 MB native app, 20 MB RAM |
| Distribution | Phase 3 not yet done | `brew install owo-network/brew/koe` works today |
| Maturity | Phase 2/7 of v1 (no install yet) | shipped, v1.0.14, regular releases |

  - Reasons to keep voice-cc: pure-local-by-default, hackability (491 lines bash+Lua), Hammerspoon ecosystem integration, sunk cost / journey value, full feature direction control.
  - Reasons to adopt Koe: it exists today, more capable, switching to local-only is one config setting, frees you for genuinely new ideas.

  Resolution requires user input. No engineering can proceed until path is chosen.
</blockers>
