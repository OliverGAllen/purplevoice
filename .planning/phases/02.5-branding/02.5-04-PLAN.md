**Name:** ` + "`PurpleVoice`" + ` (PascalCase brand). Lowercase ` + "`purplevoice`" + ` is the canonical form for the Hammerspoon module, the bash binary, and XDG paths.

Every alternative we considered (Koe, Wispr Flow, Voicy, SuperWhisper at any tier with cloud sync, the Mac's built-in Dictation set to "use server-based") sends audio off the machine by default or by subscription. PurpleVoice runs whisper.cpp locally on the Metal backend on Apple Silicon. No accounts, no API keys, no telemetry, no opt-in cloud features that could later become opt-out. Audio is captured to a temporary file, transcribed by a local binary, the WAV is deleted on every exit path, and the transcript is pasted into the focused window.

- **Privacy-conscious individuals** who don't want their voice — or its transcripts — leaving their machine
- **Government, defence, and intelligence personnel** whose data-handling policies prohibit cloud STT
- **Healthcare professionals** bound by HIPAA and equivalent privacy rules; **legal professionals** bound by attorney-client privilege
- **Finance and compliance roles** where voice content may include MNPI or regulated PII
- **Journalists** handling sensitive sources whose confidentiality is operational, not aspirational
- **Air-gapped or restricted-network operators** where cloud STT is technically impossible

# PurpleVoice

**Local voice dictation. Nothing leaves your Mac.**

![PurpleVoice](assets/icon-256.png)

A local, push-to-talk voice input system for Claude Code on macOS Apple Silicon. Hold a hotkey, speak, release — your transcript appears in the focused Claude Code terminal. Audio never leaves your Mac.

## Why PurpleVoice

PurpleVoice is built for people and organisations whose privacy requirements rule out cloud-based dictation: privacy-conscious individuals; government, defence, and intelligence personnel; healthcare and legal professionals; finance and compliance roles; journalists handling sensitive sources; and air-gapped or restricted-network environments.

Every other dictation tool we considered defaults to cloud or pushes you toward a subscription: Koe (cloud default), Wispr Flow (subscription cloud), SuperWhisper (cloud sync at any paid tier), the Mac's built-in Dictation when "Enhanced" is enabled. PurpleVoice runs whisper.cpp locally on the Metal backend. No accounts, no API keys, no telemetry, no opt-in cloud features. The audio file is captured, transcribed locally, and deleted on every exit path. The transcript is pasted into the focused window. Everything stays on the Mac.

For institutional users with strict privacy requirements: the Phase 2.7 `SECURITY.md` document (produced separately) covers the threat model, auditable zero-egress verification methodology, SBOM, and gap analysis against NIST SP 800-53 / FIPS 140-3 / FedRAMP / Common Criteria expectations. PurpleVoice's claims are auditable and verifiable, not formally certified.

## Setup

```bash
bash setup.sh
```

`setup.sh` is idempotent — safe to re-run. It:
- Installs Homebrew dependencies (Hammerspoon, sox, whisper-cpp)
- Migrates any existing `~/.config/voice-cc/`, `~/.local/share/voice-cc/`, `~/.cache/voice-cc/` directories to the `purplevoice` equivalents (one-time, idempotent)
- Creates the XDG directory layout: `~/.config/purplevoice/`, `~/.local/share/purplevoice/models/`, `~/.cache/purplevoice/`, `~/.local/bin/`, `~/.hammerspoon/purplevoice/`
- Downloads the Whisper `small.en` model with SHA256 verification (resumable)
- Downloads the Silero VAD weights (size sanity-checked)
- Seeds a default vocabulary file (`vocab.txt` — never clobbered on re-run)
- Installs the canonical hallucination denylist (always overwritten — project-owned)
- Creates symlinks: `~/.local/bin/purplevoice-record` and `~/.hammerspoon/purplevoice/`

After setup completes, paste this line into your `~/.hammerspoon/init.lua`:

```lua
require("purplevoice")
```

(If you previously had `require("voice-cc")`, replace it with the line above.)

## Permissions

PurpleVoice piggybacks on Hammerspoon's permissions — there's no separate `.app` for macOS to track.

- **Microphone** — System Settings → Privacy & Security → Microphone → enable Hammerspoon.app
- **Accessibility** — System Settings → Privacy & Security → Accessibility → enable Hammerspoon.app

Both are required. Hammerspoon prompts for them automatically. If the prompt was dismissed, PurpleVoice surfaces a follow-up notification with a deep link to the right Settings pane.

## Conflicting macOS feature to disable

Disable the macOS Dictation hotkey to avoid conflicts with `cmd+shift+e`:

- System Settings → Keyboard → Dictation → Shortcut → Off

## File layout (post-install)

```
~/.config/purplevoice/                vocab.txt + denylist.txt
~/.local/share/purplevoice/models/    ggml-small.en.bin (~488 MB) + ggml-silero-v6.2.0.bin (~885 KB)
~/.cache/purplevoice/                 (transient logs, cleaned by setup.sh / runtime)
~/.local/bin/purplevoice-record       symlink to repo
~/.hammerspoon/purplevoice/           symlink to repo
/tmp/purplevoice/                     transient WAVs (cleaned via EXIT trap on every exit)
```

## Architecture

Brief: Hammerspoon binds the hotkey, spawns `purplevoice-record` (bash) on press, sends SIGTERM on release. Bash glue runs sox to capture, whisper-cli to transcribe, prints the transcript on stdout. Hammerspoon writes the transcript to the clipboard with a transient UTI marker (so clipboard managers don't keep it), synthesises `cmd+v`, then restores the prior clipboard after 250 ms.

# test_brand_consistency.sh — Phase 2.5 regression catch.

#   - anything under .planning/ (historical record, per CONTEXT.md D-07)

#     in STATE.md to update STACK.md source-of-truth; out of scope for Phase
#     2.5 per checker revision iter 1, Option B)
#
# Source: .planning/phases/02.5-branding/02.5-VALIDATION.md "Sampling Rate -> Full suite command".
set -euo pipefail
cd "$(dirname "$0")/.."   # repo root

# Find any code/doc file containing `voice-cc`, excluding the historical record.
HITS=$(grep -rln "voice-cc" \
  --include="*.lua" --include="*.sh" --include="*.md" --include="*.txt" \
  . 2>/dev/null \
  | grep -v "^\./\.planning/" \
  | grep -v "^\./\.git/" || true)

# Filter out setup.sh (migrate FROM-args) and CLAUDE.md (auto-managed block).
HITS=$(echo "$HITS" | grep -v "^\./setup\.sh$" | grep -v "^\./CLAUDE\.md$" || true)

# Verify setup.sh has exactly 3 voice-cc migrate FROM-arg occurrences (no more, no less).
# BSD-grep safe: pattern matches the literal "voice-cc" inside a quoted "$HOME/..." path arg.
SETUP_COUNT=$(grep -c '"\$HOME/[^"]*voice-cc' setup.sh 2>/dev/null || echo 0)

if [ "$SETUP_COUNT" -ne 3 ]; then
  echo "FAIL: setup.sh has $SETUP_COUNT voice-cc path strings (expected exactly 3 — the migrate_xdg_dir FROM args)"
  exit 1
fi

if [ -n "$HITS" ]; then
  echo "FAIL: unexpected voice-cc references in user-facing files:"
  echo "$HITS"
  exit 1
fi

echo "PASS: no voice-cc strings in user-facing surfaces (setup.sh migration FROM-args verified == 3; CLAUDE.md auto-managed block exempt — Phase 3 STACK.md follow-up logged)"
exit 0
```
