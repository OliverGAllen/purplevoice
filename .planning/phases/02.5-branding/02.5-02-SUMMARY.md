## Decisions Made

- **BSD grep escape form (revision iter 1).** Verify clauses use `'$HOME/\.config/voice-cc'` form (escape only the dot, not the dollar). Verified empirically: `/usr/bin/grep` on macOS Sequoia 15.7.5 treats `$` as literal in middle of BRE pattern when followed by non-special characters; the alternative `'\$HOME/.config/voice-cc'` would be interpreted as literal-backslash-then-dollar and fail. Note: this sandbox's PATH-resolved `grep` is `ugrep 7.5.0` which has different `$`-anchor semantics — verification used absolute `/usr/bin/grep` to match user's runtime grep.

- **Migration block placed at Step 3b, BEFORE Step 4 mkdir.** Per RESEARCH Pattern 2: if Step 4 ran first, mkdir would create empty `~/.config/purplevoice`, triggering the only-new branch (no-op) on the migration block — and the user's `~/.config/voice-cc` data would be orphaned forever. Step 3b before Step 4 ensures only-old branch fires correctly.

- **`~/.hammerspoon/purplevoice` removed from mkdir block (Rule 1 deviation).** The plan instructed to include `~/.hammerspoon/purplevoice` in Step 4's mkdir. But Step 6c uses `ln -sfn` to create that path as a symlink — and `ln -sfn` cannot overwrite a directory. The mkdir-then-ln race made `~/.hammerspoon/purplevoice` an empty dir on the first live run. Auto-fix: removed from mkdir; added explanatory comment; mkdir keeps `~/.hammerspoon` (parent dir) so the symlink can be created as a child. Verified via second run: `[ -L "$HOME/.hammerspoon/purplevoice" ]` passes.

- **Live migration performed despite parallel Plan 02.5-03.** Plan 02.5-03 was concurrently editing `purplevoice-lua/init.lua`'s contents (menubar styling), but did not modify the directory structure. setup.sh only creates a symlink TO the directory — it doesn't read init.lua's content. Race-free. Both plans' final SUMMARYs exist on disk now (no merge conflict).

- **Pattern 1 (per-file targeted edit) over global sed.** Per RESEARCH Pitfall 1: a global sed pass would have rewritten BOTH sides of the migrate_xdg_dir calls, making them no-ops. Manual per-string Edit tool calls preserved the literal voice-cc FROM args.
