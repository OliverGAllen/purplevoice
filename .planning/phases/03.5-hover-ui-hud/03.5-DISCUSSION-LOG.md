## Visual Form Factor

### Q1: HUD primary form factor while recording?

| Option | Description | Selected |
|--------|-------------|----------|
| Pill (Recommended) | Lavender background + white "● Recording" text, ~140×36px, rounded-rect. Distinctive, branded, instant comprehension. | ✓ |
| Dot only | Solid lavender circle, ~32px, no text. Most minimal. | |
| Icon + text | Lips silhouette from icon-256.png scaled + "Recording" beside it. Strongest brand reinforcement. | |
| Tagline card | Wider ~220×60 with the canonical tagline + recording dot. Maximally branded. | |

**User's choice:** Pill (Recommended)

### Q2: On-HUD content during recording?

| Option | Description | Selected |
|--------|-------------|----------|
| "● Recording" (Recommended) | Filled circle glyph + word "Recording". Two-token comprehension. | ✓ |
| "●" dot only | Just the filled circle. Loses contrast since HUD only appears during recording. | |
| "● 00:00" elapsed timer | Dot + live timer. Needs hs.timer-driven redraw — contradicts HUD-03 zero-CPU-when-idle when visible. | |
| "● Recording — release to send" | Instructional. Helpful first-N-uses then verbose. | |

**User's choice:** "● Recording" (Recommended)

### Q3: Visual style for the HUD background?

| Option | Description | Selected |
|--------|-------------|----------|
| Flat lavender + white text (Recommended) | Solid #B388EB, white text, no shadow, no border. Fastest render, lowest GPU cost. | |
| Drop shadow + rounded corners | Solid lavender + 8px border-radius + soft drop shadow. More macOS-native. | |
| Glassmorphism (translucent + blur) | Translucent lavender with hs.canvas vibrancy/blur backdrop. Most Sequoia-native. Highest GPU cost; blur edge artifacts. | |
| Solid + small white outline | Solid lavender + 1.5px white border. Less brand-distinctive than pure flat. | |
| Other (free text) | "lets do 3 but without the blur" | ✓ |

**User's free-text response:** "lets do 3 but without the blur"

**Captured intent:** Translucent lavender at alpha ~0.85 (the modern semi-transparent option 3 specified) but WITHOUT the hs.canvas vibrancy/blur backdrop layer. Modern semi-transparent look + lower GPU cost + avoids text-edge artifacts from backdrop blur. Reflected back to user inline; user moved to next area.

### Q4: HUD physical size?

| Option | Description | Selected |
|--------|-------------|----------|
| Compact ~140×36 (Recommended) | Big enough to read clearly, small enough to be unobtrusive. | ✓ |
| Generous ~200×60 | Easier to spot in peripheral vision. Risk: feels like a notification, not an indicator. | |
| Tiny ~80×24 | Minimal interference; could be missed in peripheral vision. | |
| Adaptive | Sizes to content. Adds layout-logic complexity. | |

**User's choice:** Compact ~140×36 (Recommended)

## Default Position

### Q1: Where does the HUD appear by default?

| Option | Description | Selected |
|--------|-------------|----------|
| Top-center (Recommended) | Top-edge of active screen, horizontally centered, ~50px below menubar. | ✓ |
| Top-right (near menubar) | Pairs visually with the menubar lavender dot. Risk: collides with notifications dropdown. | |
| Bottom-center | Typical macOS notification location. May collide with Dock. | |
| Near cursor | Floats near mouse. Risk: covers content. | |

**User's choice:** Top-center (Recommended)

### Q2: Multi-monitor — which screen?

| Option | Description | Selected |
|--------|-------------|----------|
| Active screen (Recommended) | Screen containing the focused window. | ✓ |
| Primary screen only | Always on macOS-designated primary; predictable but may be off-attention. | |
| Screen under cursor | Useful if user has different screens for different tasks. | |

**User's choice:** Active screen (Recommended)

### Q3: Position override mechanism?

| Option | Description | Selected |
|--------|-------------|----------|
| Named positions (Recommended) | PURPLEVOICE_HUD_POSITION = top-center / top-right / bottom-center / bottom-right / near-cursor / center. | ✓ |
| Named OR explicit coords | Above + PURPLEVOICE_HUD_POSITION=x,y for absolute pixels. More code paths to test. | |
| Explicit coords only | PURPLEVOICE_HUD_X / PURPLEVOICE_HUD_Y. Most flexible, highest setup friction. | |

**User's choice:** Named positions (Recommended)

### Q4: Distance from screen edge for top-center default?

| Option | Description | Selected |
|--------|-------------|----------|
| ~50px below menubar (Recommended) | Comfortable margin; standalone visual, not bumped against menubar. | ✓ |
| Flush with top edge (just below menubar) | Tightest visual coupling. Risk: feels like menubar overflow. | |
| ~100px below menubar | More breathing room; HUD feels intentionally placed but disconnected. | |

**User's choice:** ~50px below menubar (Recommended)

## Configuration

### Q1: Default state for new users?

| Option | Description | Selected |
|--------|-------------|----------|
| On by default; PURPLEVOICE_HUD_OFF=1 disables (Recommended) | Showcases the feature; matches PURPLEVOICE_NO_SOUNDS pattern. | ✓ |
| Off by default; PURPLEVOICE_HUD=1 enables | Opt-in. Conservative. Risk: users never discover the feature. | |
| On for first 7 days, then off | Adds state (timestamp on first run); contradicts simplicity. | |

**User's choice:** On by default; PURPLEVOICE_HUD_OFF=1 disables (Recommended)

### Q2: Which env vars should the HUD expose?

| Option | Description | Selected |
|--------|-------------|----------|
| On/off + position only (Recommended) | PURPLEVOICE_HUD_OFF + PURPLEVOICE_HUD_POSITION. Two knobs. | ✓ |
| On/off + position + alpha | Above + PURPLEVOICE_HUD_ALPHA. One more knob. | |
| On/off + position + alpha + offset | Above + per-position offsets. Power-user flexibility. | |
| Comprehensive (all visual properties) | Risk: brand drift (users override lavender to anything); contradicts Phase 2.5 brand lock. | |

**User's choice:** On/off + position only (Recommended)

### Q3: Runtime toggle hotkey?

| Option | Description | Selected |
|--------|-------------|----------|
| No runtime toggle (Recommended) | Env var read once at module load. Reload Hammerspoon to apply. Simpler. | ✓ |
| cmd+shift+h to toggle | Risk: collision with macOS Hide. | |
| cmd+option+shift+h | Avoids cmd+H collision; harder to remember. | |

**User's choice:** No runtime toggle (Recommended)

### Q4: setup.sh banner integration?

| Option | Description | Selected |
|--------|-------------|----------|
| Brief one-liner (Recommended) | "HUD options: PURPLEVOICE_HUD_OFF=1 to disable, PURPLEVOICE_HUD_POSITION=top-right etc." | ✓ |
| Full env var documentation block | Multi-line wall of text. | |
| Don't mention in banner; document in README only | Risk: users miss the feature. | |
| Brief one-liner + README section | Both. Slight duplication. | |

**User's choice:** Brief one-liner (Recommended)

## Screen-Recording Visibility

### Q1: Should the HUD appear in screen recordings by default?

| Option | Description | Selected |
|--------|-------------|----------|
| Hidden by default (Recommended) | NSWindowSharingNone or Hammerspoon equivalent. Matches "Local voice dictation. Nothing leaves your Mac." ethos. | ✓ |
| Visible by default | Standard NSWindow sharing. Simple and predictable. | |
| Configurable (PURPLEVOICE_HUD_VISIBLE_IN_RECORDINGS) | Third env var; default hidden, env var flips to visible. | |

**User's choice:** Hidden by default (Recommended)

### Q2: If macOS Sequoia 15.7.5 hides-from-recording API behaves unexpectedly, framing?

| Option | Description | Selected |
|--------|-------------|----------|
| Honest caveat in README + SECURITY.md (Recommended) | "PurpleVoice attempts to hide the HUD; macOS-version behaviour may vary; PURPLEVOICE_HUD_OFF=1 for absolute privacy." Best-effort framing. | ✓ |
| Fail-closed: refuse to show HUD if hide API unavailable | Strict; some users get no HUD silently. | |
| Don't claim hides-from-recordings at all | Drop the feature; default visible. Loses privacy-first selling point. | |

**User's choice:** Honest caveat in README + SECURITY.md (Recommended)
