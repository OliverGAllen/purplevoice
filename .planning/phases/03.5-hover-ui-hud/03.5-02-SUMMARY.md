| 1 | Live verification methodology | `launchctl setenv PURPLEVOICE_HUD_POSITION top-right` + `hs.reload()` does NOT propagate to Hammerspoon's `os.getenv` view. Lua's `os.getenv` is a thin wrapper around C's `getenv()`, which reads the process's `environ` array — set at exec time, not refreshed on Lua re-init. Hammerspoon was already running before the env var was set, so `os.getenv("PURPLEVOICE_HUD_POSITION")` returned nil regardless of `launchctl setenv` calls. | Documented for future testing: to verify env-var-driven position changes live, fully quit Hammerspoon (cmd+Q) and relaunch from a shell with the env var pre-set: `PURPLEVOICE_HUD_POSITION=bottom-right open -a Hammerspoon`. The README ## Security & Privacy HUD subsection (Plan 03 territory) should mention this pathway in the configuration documentation. |
