## Decisions Made

1. **Syft install is unconditional, NOT under OFFLINE branching** — per checker M-3 Option A. Phase 2.7 makes Syft a hard dependency rather than introducing a Syft-absent skip path in verify_sbom.sh. SEC-03 substantiation requires a real Syft scan asserting `name == "PurpleVoice"`; the placeholder SBOM would fail that assertion if Syft were absent at execution time. Cleaner to require Syft.
2. **DESIGN NOTE block in process_tree.sh** explains why the helper writes to the production WAV path `/tmp/purplevoice/recording.wav`: (a) Lua isRecording single-instance guard prevents concurrent invocations; (b) `PURPLEVOICE_TEST_SKIP_SOX=1` is set unconditionally in this test path so sox does NOT spawn (no risk of overwriting the pre-staged WAV); (c) purplevoice-record exposes no path-override env var, and adding one would require modifying the Pattern 2 boundary file. If a future revision exposes such an env var, the helper switches to `/tmp/purplevoice/test-recording.wav`.
3. **`\bcompliant\b` / `\bcertified\b` / `\bguarantees\b` baseline clean** in SECURITY.md — even the framing-explainer paragraph at the top of the document avoids these words (the plan's prose used them, but the verify regex banned them; the verify regex wins, see Deviation 1 below). Plan 02.7-01's lint enforces this baseline going forward.
