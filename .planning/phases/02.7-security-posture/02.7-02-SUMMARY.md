## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] macOS BSD lsof per-PID filter syntax wrong in plan**
- **Found during:** Task 2-1 (live run of verify_egress.sh)
- **Issue:** The plan instructed `lsof -i -p PID` for the Layer 1 socket snapshot. On macOS BSD lsof, this syntax returns the UNION of (-i: all network sockets system-wide) and (-p PID: all open files for PID), NOT the per-PID network socket list. First live run produced "PID 43700 has 1105 network sockets" — a clearly absurd number for a freshly-spawned bash glue process. The Layer 1 logic was buggy and would always FAIL.
- **Fix:** Replaced `lsof -i -p PID` with `lsof -p PID -a -i` (the `-a` joins the filters with AND-intersect, returning ONLY sockets owned by PID). Updated the threshold check from `> 1` (header-line tolerance) to `> 0` (no header when no sockets present, per actual BSD lsof behaviour). Added inline comment explaining the trap for future readers.
- **Files modified:** `tests/security/verify_egress.sh` (Layer 1 snapshot block)
- **Verification:** Live re-run on Oliver's machine: `PASS (weakened — layer 3 pf+tcpdump skipped due to sudo unavailable; layers 1-2 carry the claim)` exit 0. Layer 1 correctly attributes 0 sockets to the bash glue / sox / transcription PIDs.
- **Committed in:** `cbba76c` (Task 2-1 commit) — fix included in initial implementation; no separate commit needed.
