## Decisions Made

1. **Per-control table extended to 20 rows (vs the 15 minimum).** RESEARCH Priority 4 lists 16 preview controls (AC-3, AC-6, AC-14, AU-2, AU-12, IA-2, IA-5, SC-7, SC-8, SC-12, SC-28, SI-2, SI-7, SI-10, PT-3, PT-4); the plan's `<interfaces>` block extends this to 20 by adding CM-7 (Least Functionality), MP-6 (Media Sanitization), SA-15 (Development Process), and SR-3 (Supply Chain Controls). I included all 20 because the LIMITED families (CM, MP, SA, SR per the Applicability Matrix) deserve representation alongside the Yes-applies families — otherwise the Per-Control Mapping table is biased toward Yes-applies and reads as if Limited-applies families had nothing to say.

2. **AU-2 / AU-12 'Not Pursued' framing anchored to STRIDE §Repudiation cross-reference.** Rather than restating the privacy > accountability trade-off independently in the AU rows, both rows cross-reference the §Threat Model STRIDE Repudiation row (Plan 01 territory). This gives auditors a single load-bearing decision-source ("see Repudiation row") rather than two separate justifications that could drift apart on future edits.

3. **AC-6 verification command keeps literal `whisper-cli` binary name.** The substantiation column says "Verified by `ls -l /opt/homebrew/bin/sox /opt/homebrew/bin/whisper-cli`" — an auditor needs a concrete copy-pasteable command. The narrative cell prose still uses the abstract phrasing "the transcription binary run as the user UID". This pattern (abstract prose + concrete verification command) is consistent with the existing SECURITY.md §Egress Verification anchor: prose says "transcription binary"; verify_egress.sh names purplevoice-record as the boundary file. The Pattern 2 corollary discipline applies to long-lived narrative prose, not to verification commands an auditor must literally type.

4. **10 organisational families listed Out of Scope explicitly with rationale.** D-14 calls for skipping organisational families. Listing them silently would leave an auditor wondering "did the author know these exist?". Listing them explicitly with rationale ("organisational; not relevant to a personal tool") is auditor-honest framing per RESEARCH Priority 4 honesty discipline.

5. **`Met (via OS)` qualifier on IA-2 row.** macOS user session = identification; PurpleVoice inherits. The qualifier `(via OS)` makes the inheritance explicit rather than over-claiming PurpleVoice itself as the identification provider. Same honesty discipline as the family-level Out-of-Scope rationale.
