## Manual-Only Verifications

| Behavior | Requirement | Why Manual | Test Instructions |
|----------|-------------|------------|-------------------|
| pf rule enforcement on macOS Sequoia 15.7.5 (build 24G624 specifically) | SEC-02 (caveat) | Public sources confirm regression on 15.0-15.3.1; status on 15.7.5 unconfirmed. Positive-control in `verify_egress.sh` detects at test-time, but a manual pre-release sweep on Oliver's exact macOS build confirms whether the strongest layer (pf+tcpdump) is producing real evidence vs falling back to lsof/nettop layers. | (1) Run `bash tests/security/verify_egress.sh` (2) Read the positive-control output line: PASS = pf enforcing; WARN = pf bypassed, lsof/nettop layers carry the claim (3) If WARN, manually update SECURITY.md "Egress Verification" section to reflect that on this macOS build, the egress claim rests on socket-state evidence (lsof + nettop) rather than packet-flow evidence (pf + tcpdump). |
| NIST 800-53 control mapping rows (auditor judgment) | SEC-01 (gap analysis depth) | Per-control "Met / Partial / Not Pursued / N/A" rows are interpretive — a sysadmin / auditor reading SECURITY.md may disagree with researcher's mapping. Pre-release sweep by Oliver (or external review) catches over-claims. | Read each row in SECURITY.md §"NIST 800-53 Rev 5 / Low-baseline Mapping". For each "Met" claim, confirm a verification script or codebase reference substantiates. For each "Partial", confirm the partial framing is honest. Flag any row that smells like an over-claim. |
| "Compatible with" framing across all 6 framed framework sections (FIPS / FedRAMP / CC / HIPAA / SOC 2 / ISO 27001) | D-17 (mandatory framing constraint) | Lint catches obvious "compliant" / "certified" / "guarantees" leaks. Flag any sentence that an institutional buyer could screenshot to claim "PurpleVoice is HIPAA / SOC 2 / ISO 27001 compliant". Reword to "supports" / "compatible with" / "consistent with applicable obligations for in-scope code". |
| Air-gap install path on a genuinely offline machine | SEC-06 (substantiation) | Automated test in `verify_air_gap.sh` simulates offline mode but doesn't prove the documented procedure works in a real air-gapped environment. | Optional pre-release: on a separate Mac, disconnect from network, run `PURPLEVOICE_OFFLINE=1 bash setup.sh` after manually placing ggml-small.en.bin at the documented path. If unable to source a separate offline machine, document the pre-release check as "deferred to first institutional adopter feedback". |
